# 🔍 檢查文件位置

## 問題診斷

如果 `http://localhost:8000/article-cms.js` 返回 404，說明：

1. **服務器沒有在項目根目錄啟動**
2. **或者文件不在正確位置**

## ✅ 解決方案

### 步驟 1：確認文件位置

確認以下文件在**項目根目錄**（包含 `site1`, `site2` 等資料夾的目錄）：
- `article-cms.js`
- `home-cms.js`

### 步驟 2：在項目根目錄啟動服務器

**重要**：必須在項目根目錄啟動服務器！

```bash
# 1. 切換到項目根目錄
cd "你的項目路徑"

# 2. 確認文件存在
dir article-cms.js
dir home-cms.js

# 3. 啟動服務器
python -m http.server 8000
```

### 步驟 3：測試

訪問以下 URL，應該都能看到文件內容：
- `http://localhost:8000/article-cms.js` ✅
- `http://localhost:8000/home-cms.js` ✅
- `http://localhost:8000/site1/index.html` ✅
- `http://localhost:8000/site2/index.html` ✅

## 🔍 為什麼 site1 可以但 site2-5 不行？

如果 site1 可以，說明：
- 服務器確實在項目根目錄啟動
- 文件確實存在

那 site2-5 應該也可以。如果不行，可能是：
1. **瀏覽器緩存**：清除緩存或強制刷新（Ctrl+F5）
2. **路徑解析問題**：檢查瀏覽器 Network 標籤，看實際請求的 URL

## 💡 快速測試

1. 訪問：`http://localhost:8000/article-cms.js`
2. 如果看到 JavaScript 代碼 → ✅ 文件存在，路徑正確
3. 如果 404 → ❌ 服務器不在項目根目錄，或文件不存在
