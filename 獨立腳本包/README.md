# 📦 獨立腳本包使用說明

## 🎯 功能說明

這個腳本包可以讓您：
- ✅ **不需要下載整個專案目錄**
- ✅ **可以在任何位置建立網站資料夾**
- ✅ **自動上傳到 Strapi 和 GitHub**

---

## 📋 檔案說明

### 包含的檔案

1. **`上傳網站.bat`** - 主批次檔（一鍵執行）
2. **`upload-site-to-strapi.js`** - 上傳腳本（支援任何網站名稱）

---

## 🚀 使用方式

### 步驟 1：下載腳本包

**只需要下載這兩個檔案：**
- `上傳網站.bat`
- `upload-site-to-strapi.js`

**放在任何位置都可以：**
```
桌面/
└── 上傳腳本/
    ├── 上傳網站.bat
    └── upload-site6-10-to-strapi.js
```

### 步驟 2：準備網站資料夾

**可以在任何位置建立網站：**
```
桌面/
└── site6/              ← 您的網站（可以在任何位置）
    ├── index.html
    ├── about.html
    ├── articles/
    │   └── 2025-12-01.html
    ├── css/
    │   └── style.css
    └── js/
        └── main.js
```

### 步驟 3：執行腳本

```bash
# 雙擊執行
上傳網站.bat
```

**按照提示：**
1. 輸入網站資料夾路徑（例如：`C:\Users\...\桌面\site6`）
2. 選擇是否已有專案目錄
   - **沒有** → 腳本會自動 Clone GitHub 倉庫
   - **有** → 輸入專案目錄路徑
3. 腳本會自動處理所有步驟

### 步驟 4：完成

- ✅ 網站檔案推送到 GitHub
- ✅ 內容上傳到 Strapi
- ✅ Vercel 自動部署

---

## 🔄 自動化流程

### 完整流程

```
1. 選擇網站資料夾（任何位置）
   ↓
2. 準備 Git 工作區
   - 如果沒有專案目錄 → 自動 Clone
   - 如果有專案目錄 → 使用現有的
   ↓
3. 複製網站檔案到專案目錄
   ↓
4. 提取 HTML 內容並轉換成 Strapi 格式
   ↓
5. 上傳到 Strapi
   ↓
6. 推送到 GitHub
   ↓
7. Vercel 自動部署
```

### 不需要下載整個專案

**傳統方式：**
- ❌ 需要下載整個專案（幾百 MB）
- ❌ 需要保持專案目錄同步

**使用獨立腳本：**
- ✅ 只需要下載 2 個檔案（幾 KB）
- ✅ 可以在任何位置建立網站
- ✅ 腳本自動處理所有步驟

### 靈活使用

**可以在任何位置：**
- 桌面
- 我的文件
- 其他資料夾
- 任何位置

**腳本會自動：**
- Clone 專案（如果需要）
- 複製檔案到專案目錄
- 上傳到 Strapi
- 推送到 GitHub
