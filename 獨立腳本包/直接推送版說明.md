# 🚀 直接推送版說明

## 🎯 功能說明

`上傳網站-直接推送版.bat` 是一個**真正輕量**的上傳腳本，特點是：

- ✅ **不需要下載根目錄**
- ✅ **直接連接到 GitHub 倉庫**
- ✅ **只推送您的檔案**
- ✅ **不會影響其他檔案**

---

## 💡 工作原理

### 傳統方式 vs 直接推送

**傳統方式：**
```
1. Clone 整個倉庫（幾百 MB）❌
2. 複製檔案
3. 推送
```

**直接推送版：**
```
1. 連接 GitHub 倉庫 ✅
2. 只準備您的檔案
3. 直接推送
```

---

## 🚀 使用方式

### 步驟 1：準備網站資料夾

**可以在任何位置建立網站：**
```
桌面/
└── site11/              ← 您的網站（可以是任何名稱）
    ├── index.html
    ├── about.html
    ├── articles/
    │   └── 2025-12-01.html
    ├── css/
    │   └── style.css
    └── js/
        └── main.js
```

### 步驟 2：執行直接推送版腳本

```bash
# 雙擊執行
上傳網站-直接推送版.bat
```

**按照提示：**
1. 輸入網站資料夾路徑（例如：`C:\Users\...\桌面\site11`）
2. 腳本會自動：
   - 連接 GitHub 倉庫
   - 準備您的檔案
   - 上傳到 Strapi
   - 直接推送到 GitHub

### 步驟 3：完成

- ✅ 網站檔案推送到 GitHub（只推送您的檔案）
- ✅ 內容上傳到 Strapi
- ✅ Vercel 自動部署

## 🔄 工作流程

### 直接推送流程

```
1. 選擇網站資料夾（任何位置）
   ↓
2. 建立臨時目錄
   ↓
3. 初始化 Git 倉庫
   ↓
4. 連接 GitHub 倉庫（不下載檔案）
   ↓
5. 複製您的網站檔案
   ↓
6. 上傳到 Strapi
   ↓
7. 直接推送到 GitHub（只推送您的檔案）
   ↓
8. Vercel 自動部署
```

## 💡 重要說明

### 不會下載根目錄

**直接推送版：**
- ✅ 只連接 GitHub 倉庫
- ✅ 只準備您的檔案
- ✅ 只推送您的檔案
- ❌ 不會下載其他檔案

### 不會影響其他檔案

**GitHub 倉庫結構：**
```
multi-site-static-strapi-front/
├── site1/      ← 不會影響
├── site2/      ← 不會影響
├── site3/      ← 不會影響
├── ...
├── site10/     ← 不會影響
└── site11/     ← 您新增的網站（只推送這個）
```

**只會：**
- ✅ 新增或更新 `site11/` 資料夾
- ✅ 不會影響其他資料夾
