# 🪶 輕量版上傳腳本說明

## 🎯 功能說明

`上傳網站-輕量版.bat` 是一個**輕量版本**的上傳腳本，特點是：

- ✅ **不需要下載整個根目錄**
- ✅ **只下載需要的檔案**
- ✅ **只推送變更的部分**
- ✅ **不會影響 GitHub 倉庫中的其他檔案**

---

## 💡 為什麼需要輕量版？

### 傳統方式的問題

**完整 Clone：**
- ❌ 需要下載整個倉庫（幾百 MB）
- ❌ 包含所有網站（site1-site10 等）
- ❌ 下載時間長
- ❌ 佔用空間大

### 輕量版的優勢

**Sparse Checkout：**
- ✅ 只下載需要的檔案
- ✅ 只下載您要上傳的網站資料夾
- ✅ 下載速度快
- ✅ 佔用空間小

---

## 🚀 使用方式

### 步驟 1：準備網站資料夾

**可以在任何位置建立網站：**
```
桌面/
└── site11/              ← 您的網站（可以是任何名稱）
    ├── index.html
    ├── about.html
    ├── articles/
    │   └── 2025-12-01.html
    ├── css/
    │   └── style.css
    └── js/
        └── main.js
```

### 步驟 2：執行輕量版腳本

```bash
# 雙擊執行
上傳網站-輕量版.bat
```

**按照提示：**
1. 輸入網站資料夾路徑（例如：`C:\Users\...\桌面\site11`）
2. 腳本會自動：
   - 只下載需要的檔案
   - 複製您的網站檔案
   - 上傳到 Strapi
   - 只推送變更的部分到 GitHub

### 步驟 3：完成

- ✅ 網站檔案推送到 GitHub
- ✅ 內容上傳到 Strapi
- ✅ Vercel 自動部署

---

## 🔄 工作流程

### 輕量模式流程

```
1. 選擇網站資料夾（任何位置）
   ↓
2. 建立臨時目錄
   ↓
3. 初始化 Git 倉庫
   ↓
4. 設定 Sparse Checkout（只下載需要的檔案）
   - 只下載：site11/
   - 只下載：upload-site-to-strapi.js
   - 不下載：其他網站（site1-site10）
   ↓
5. 複製您的網站檔案
   ↓
6. 上傳到 Strapi
   ↓
7. 只推送變更的部分到 GitHub
   ↓
8. Vercel 自動部署
```

## 📊 對比

### 完整版 vs 輕量版

| 項目 | 完整版 | 輕量版 |
|------|--------|--------|
| 下載大小 | 幾百 MB | 幾 MB |
| 下載時間 | 較長 | 較短 |
| 佔用空間 | 較大 | 較小 |
| 推送內容 | 整個倉庫 | 只推送變更 |
| 適用場景 | 需要完整倉庫 | 只上傳單一網站 |

---

## 💡 重要說明

### 不會影響其他檔案

**輕量版只會：**
- ✅ 下載您要上傳的網站資料夾
- ✅ 下載必要的腳本檔案
- ✅ 推送您新增/更新的網站

**不會：**
- ❌ 下載其他網站（site1-site10）
- ❌ 修改其他網站的檔案
- ❌ 影響 GitHub 倉庫中的其他內容

### GitHub 倉庫結構

**GitHub 倉庫結構是固定的：**
```
multi-site-static-strapi-front/
├── site1/
├── site2/
├── site3/
├── ...
├── site10/
└── site11/  ← 您新增的網站
```

**輕量版只會：**
- 新增或更新 `site11/` 資料夾
- 不會影響其他資料夾

✅ **需要完整的開發環境**
