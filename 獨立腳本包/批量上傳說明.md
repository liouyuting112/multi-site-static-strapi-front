# 📦 批量上傳腳本說明

## 🎯 功能說明

`批量上傳-多網站.bat` 是一個**批量處理腳本**，可以：

- ✅ **一次處理多個網站資料夾**
- ✅ **自動掃描父資料夾中的所有網站**
- ✅ **自動轉換格式並上傳到 Strapi**
- ✅ **推送到 GitHub**
- ✅ **不會影響 site1-5**

---

## 📋 使用方式

### 步驟 1：準備資料夾結構

**父資料夾結構：**
```
網站資料夾/
├── site6/
│   ├── index.html
│   ├── about.html
│   ├── contact.html
│   ├── privacy.html
│   └── articles/
│       ├── 2025-12-01.html
│       └── article-name.html
├── site7/
│   ├── index.html
│   └── articles/
│       └── ...
└── site8/
    └── ...
```

### 步驟 2：執行批量上傳腳本

```bash
# 雙擊執行
批量上傳-多網站.bat
```

**按照提示：**
1. 設定 GitHub 倉庫（第一次使用）
2. 設定 Strapi 後台（第一次使用）
3. 選擇包含多個網站資料夾的父資料夾
4. 腳本會自動掃描並處理所有網站

### 步驟 3：完成

- ✅ 所有網站已上傳到 Strapi（Post 和 Page）
- ✅ 所有網站已推送到 GitHub
- ✅ Vercel 自動部署

---

## 🔄 工作流程

### 批量處理流程

```
1. 設定 GitHub 和 Strapi（第一次使用）
   ↓
2. 選擇父資料夾
   ↓
3. 自動掃描所有網站資料夾
   - 檢查是否有 index.html
   - 列出所有找到的網站
   ↓
4. 確認要上傳的網站
   ↓
5. 準備 Git 工作區
   ↓
6. 批量處理每個網站：
   - 複製檔案
   - 上傳到 Strapi（Post 和 Page）
   ↓
7. 推送到 GitHub
   ↓
8. Vercel 自動部署
```

---

## 📊 處理內容

### Pages（頁面）

**每個網站會上傳：**
- `index.html` → Page type: `home`
- `about.html` → Page type: `about`
- `contact.html` → Page type: `contact`
- `privacy.html` → Page type: `privacy`

### Posts（文章）

**每個網站的 `articles/` 資料夾：**
- 日期格式（`YYYY-MM-DD.html`）→ `daily` 類別
- 其他格式 → `fixed` 類別

---

## 💡 重要說明

### 不會影響 site1-5

**腳本只會：**
- ✅ 處理您選擇的父資料夾中的網站
- ✅ 只上傳新網站或更新現有網站
- ✅ 不會修改 site1-5 的任何內容

**GitHub 倉庫結構：**
```
multi-site-static-strapi-front/
├── site1/      ← 不會影響
├── site2/      ← 不會影響
├── site3/      ← 不會影響
├── site4/      ← 不會影響
├── site5/      ← 不會影響
├── site6/      ← 您新增的網站
├── site7/      ← 您新增的網站
└── site8/      ← 您新增的網站
```

### 自動格式轉換

**腳本會自動：**
- ✅ 提取 HTML 內容
- ✅ 提取標題、圖片等
- ✅ 限制 html 欄位為 250 字元
- ✅ 自動判斷 daily/fixed 類別
- ✅ 轉換成 Strapi 格式
