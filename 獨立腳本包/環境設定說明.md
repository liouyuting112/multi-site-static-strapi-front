# Strapi 多環境設定說明

## 環境說明

### 1. 開發環境 (Development)
- **用途**：本地開發和測試
- **特點**：可以測試較長的文字內容，不受正式環境限制
- **建議**：使用本地 Strapi 或 Strapi Cloud 免費版測試專案

### 2. 測試環境 (Staging)
- **用途**：上線前的最終測試
- **特點**：模擬正式環境，但可以測試較長內容
- **建議**：使用 Strapi Cloud 的測試專案

### 3. 正式環境 (Production)
- **用途**：正式上線使用
- **特點**：有文字長度限制（250字元）
- **建議**：使用 Strapi Cloud 正式專案

## 設置步驟

### 步驟 1：創建多個 Strapi 專案

1. **開發環境**
   - 登入 Strapi Cloud
   - 創建新專案：`your-project-dev`
   - 記錄 URL 和 API Token

2. **測試環境**
   - 登入 Strapi Cloud
   - 創建新專案：`your-project-staging`
   - 記錄 URL 和 API Token

3. **正式環境**
   - 使用現有的正式專案
   - 記錄 URL 和 API Token

### 步驟 2：設定環境配置

在 `上傳設定.txt` 中設定不同環境的配置：

```txt
# 開發環境
DEV_STRAPI_URL=https://your-dev-project.strapiapp.com
DEV_STRAPI_TOKEN=your-dev-token

# 測試環境
STAGING_STRAPI_URL=https://your-staging-project.strapiapp.com
STAGING_STRAPI_TOKEN=your-staging-token

# 正式環境
PROD_STRAPI_URL=https://effortless-whisper-83765d99df.strapiapp.com
PROD_STRAPI_TOKEN=your-prod-token
```

### 步驟 3：使用腳本選擇環境

執行 `批量上傳-修正版.bat` 時，會提示選擇環境：
- `1` - 開發環境 (Development)
- `2` - 測試環境 (Staging)
- `3` - 正式環境 (Production)

## 注意事項

1. **文字長度限制**
   - 開發/測試環境：可以測試較長內容
   - 正式環境：限制 250 字元

2. **資料同步**
   - 不同環境的資料是獨立的
   - 需要分別上傳到不同環境

3. **前端配置**
   - 前端腳本 (`home-cms.js`) 需要根據部署環境使用對應的 Strapi URL
   - 可以使用環境變數或建置時設定

## 推薦工作流程

1. **開發階段**
   - 使用開發環境測試
   - 可以測試較長的文字內容

2. **測試階段**
   - 使用測試環境
   - 模擬正式環境進行測試

3. **上線階段**
   - 使用正式環境
   - 確保內容符合長度限制
