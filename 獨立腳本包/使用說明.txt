========================================
獨立腳本包 - 使用說明
========================================

🎯 功能說明
-----------
這個腳本包可以讓您：
- ✅ 不需要下載整個專案目錄
- ✅ 可以在任何位置建立網站資料夾
- ✅ 自動上傳到 Strapi 和 GitHub

📋 檔案說明
-----------
1. 上傳網站.bat - 主批次檔（一鍵執行）
2. upload-site-to-strapi.js - 上傳腳本（支援任何網站名稱）

🚀 使用方式
-----------

步驟 1：下載腳本包
只需要下載這兩個檔案：
- 上傳網站.bat
- upload-site-to-strapi.js

放在任何位置都可以，例如：
桌面/上傳腳本/
├── 上傳網站.bat
└── upload-site-to-strapi.js

步驟 2：準備網站資料夾
可以在任何位置建立網站，網站名稱可以是任何名稱：
桌面/site6/          ← 可以是 site6, site11, my-website, 任何名稱
├── index.html
├── about.html
├── articles/
│   └── 2025-12-01.html
├── css/
│   └── style.css
└── js/
    └── main.js

步驟 3：執行腳本
雙擊執行：上傳網站.bat

按照提示：
1. 輸入網站資料夾路徑（例如：C:\Users\...\桌面\site6 或任何名稱）
2. 選擇是否已有專案目錄
   - 沒有 → 腳本會自動 Clone GitHub 倉庫
   - 有 → 輸入專案目錄路徑
3. 腳本會自動處理所有步驟

步驟 4：完成
- ✅ 網站檔案推送到 GitHub
- ✅ 內容上傳到 Strapi
- ✅ Vercel 自動部署

💡 優點
--------
✅ 只需要下載 2 個檔案（幾 KB）
✅ 可以在任何位置建立網站
✅ 腳本自動處理所有步驟
✅ 不需要下載整個專案（幾百 MB）

📝 使用範例
-----------

範例 1：第一次使用
1. 下載腳本包（2 個檔案）
   放到：桌面/上傳腳本/

2. 建立網站資料夾
   位置：桌面/site6/

3. 執行腳本
   雙擊：桌面/上傳腳本/上傳網站.bat

4. 輸入網站資料夾路徑
   C:\Users\...\桌面\site6（或任何網站名稱）

5. 選擇「沒有專案目錄」
   腳本會自動 Clone

6. 完成！

範例 2：之後使用（已有專案目錄）
1. 執行腳本
   雙擊：上傳網站.bat

2. 輸入網站資料夾路徑
   C:\Users\...\桌面\site7（或任何網站名稱）

3. 選擇「已有專案目錄」
   輸入：C:\Users\...\專案目錄

4. 完成！
