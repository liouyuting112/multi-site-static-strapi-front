# AI 文章生成管理系統 - Web 介面

## 📋 功能特色

✅ **簡單易用的 Web 介面** - 非技術人員也能輕鬆操作  
✅ **多網站選擇** - 可同時選擇多個網站生成文章  
✅ **彈性設定** - 可設定每個網站生成幾篇文章（1-5 篇）  
✅ **日期選擇** - 可選擇要生成哪一天的文章  
✅ **即時回饋** - 顯示生成結果和統計資訊  

---

## 🚀 快速開始

### 步驟 1：安裝依賴

在 `文章生成管理介面` 資料夾中執行：

```bash
npm install
```

### 步驟 2：啟動伺服器

```bash
npm start
```

### 步驟 3：打開瀏覽器

訪問：`http://localhost:3000`

---

## 📖 使用說明

### 1. 選擇網站
- 勾選要生成文章的網站（可多選）
- 點擊「全選 / 取消全選」快速選擇

### 2. 設定參數
- **文章數量**：每個網站要生成幾篇文章（1-5 篇）
- **文章類別**：選擇 daily（每日）或 fixed（固定）
- **文章日期**：選擇要生成哪一天的文章（預設今天）

### 3. 開始生成
- 點擊「🚀 開始生成文章」按鈕
- 等待生成完成（會顯示進度）
- 查看結果統計

## ⚙️ 設定

### 環境變數（可選）

在 `server.js` 中修改，或設定環境變數：

```bash
set STRAPI_URL=http://localhost:1337
set STRAPI_TOKEN=你的Token
set GEMINI_API_KEY=你的API Key
```

### 修改預設值

編輯 `server.js` 中的 `CONFIG` 物件：

```javascript
const CONFIG = {
    STRAPI_URL: 'http://localhost:1337',
    STRAPI_TOKEN: '你的Token',
    GEMINI_API_KEY: '你的API Key',
    // ...
};
```

## 🎯 使用情境

### 情境 1：每天生成新文章
1. 選擇所有網站（或特定網站）
2. 設定每個網站生成 1 篇
3. 日期選擇今天
4. 點擊生成

### 情境 2：批量補齊文章
1. 選擇需要補齊的網站
2. 設定每個網站生成 2-3 篇
3. 日期選擇過去的日期
4. 點擊生成

### 情境 3：特定網站更新
1. 只選擇需要更新的網站
2. 設定生成數量
3. 選擇日期
4. 點擊生成

## ❓ 常見問題

### Q: 無法連接到後端？
**A:** 確認伺服器是否正在運行（`npm start`）

### Q: 找不到網站列表？
**A:** 確認 Strapi 是否正在運行，或檢查 `server.js` 中的 Token 設定

### Q: 生成失敗？
**A:** 檢查：
- Strapi 是否正在運行
- Token 和 API Key 是否正確
- 腳本路徑是否正確

## 🔒 安全建議

1. **不要將 Token/API Key 提交到 Git**
   - 使用環境變數
   - 或建立 `.env` 檔案（記得加入 `.gitignore`）

2. **限制訪問**
   - 可以加入認證機制
   - 或限制只能從特定 IP 訪問
